ERROR

allprojects/subprojects Blocks Detected (Configuration Anti-Pattern)

Why This Matters

Root build.gradle.kts uses allprojects{} or subprojects{} blocks which force eager configuration of all modules and are incompatible with Configuration Cache and Isolated Projects.

Production Impact

Forces Gradle to configure every module even when only one is being built. Blocks migration to Configuration Cache and the upcoming Isolated Projects feature in Gradle 9.x.

Resolution 
Replace with convention plugins applied via plugins{} block in each module, or use a shared build-logic/ included build with precompiled script plugins.
💰 Enables Configuration Cache, reduces configuration time by 50-90%, and prepares for Gradle 9.x Isolated Projects. ⏱️ Save 2.0h/cycle 📄 build.gradle.kts
INFO

Dependency Locking Not Configured

Why This Matters

No dependency locking is configured. Without locking, builds may resolve different transitive versions over time, leading to non-reproducible builds.

Production Impact

A transitive dependency could be silently upgraded to a compromised version between builds. Dependency locking ensures byte-for-byte reproducible resolution.

Resolution 
Enable locking: dependencyLocking { lockAllConfigurations() } and commit lockfiles to VCS. Run ./gradlew dependencies --write-locks to generate initial lockfiles.
💰 Reproducible builds, supply chain security, deterministic CI/CD. ⏱️ Save 0.1h/cycle 📄 build.gradle.kts
INFO

JDK Toolchain Not Configured

Why This Matters

No JDK toolchain is configured. Builds may use different JDK versions across developer machines and CI, leading to inconsistent bytecode and potential compatibility issues.

Production Impact

Different JDK versions produce different bytecode optimizations. Without toolchain pinning, a developer on JDK 21 may produce incompatible artifacts for a JDK 17 target.

Resolution 
Add kotlin { jvmToolchain(17) } or java { toolchain { languageVersion.set(JavaLanguageVersion.of(17)) } } to your build file.
💰 Consistent builds across all environments, automatic JDK provisioning. ⏱️ Save 0.1h/cycle 📄 build.gradle.kts

No Test Sources Found (Dark Module)

Why This Matters

Module 'nowinandroid' has no test files in src/test, src/androidTest, or src/commonTest. This module is completely untested.

Production Impact

Untested modules are invisible risk vectors. Bugs ship silently, regressions go undetected, and refactoring becomes dangerous. In large projects, dark modules become unmaintainable over time.

Resolution 
Add at least unit tests for public API classes. Create src/test/kotlin/ directory and add basic tests using JUnit5 or KotlinTest. Target >60% line coverage for business logic modules.
💰 Early bug detection, safe refactoring, reduced QA cycle time. Each caught regression saves 2-8h of developer time. ⏱️ Save 2.0h/cycle
